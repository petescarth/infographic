The Friction Audit: Map vs. Dashboard

In a map-first world, a simple question like "Which paddock has the least cover?" requires a multi-step cognitive process: Panning, Zooming, Clicking, and Mentally Storing values. A Dashboard reduces this to a single glance.

Task: Identify Bottom 3 Paddocks

The "Map Tax"

Spatial interfaces require high motor-control (pan/zoom) and working memory. On a mobile device in a bouncing truck, this friction increases by 400% compared to a static list.

The "Dashboard Dividend"

The Actionability Matrix

Not all data is created equal. The current interface treats every pixel with the same weight. We must prioritize data based on its "Decisional Gravity."

01
Total Farm Biomass

The "Fuel Gauge." This is the first number a farmer needs. If it's green, they might not even open the map.

02
Days of Feed Remaining

Calculated metadata. Converting TSDM into a "Time" metric is the ultimate friction reducer.

03
Spatial Anomalies

This is where the map shines. Don't show the map for totals; show it for anomalies (e.g., specific fenceline pressure).

Recommendation A

Introduce a "Summary Landing Page". Stop the auto-load of the Map GL engine. Instead, load a lightweight JSON dashboard that displays Farm Totals, Trends, and Rainfall stats first.

Recommendation B

Implement "Threshold Alerts". Allow farmers to set a "Panic Level" for TSDM. When satellite data hits that level, send a push notification that links directly to a paddock summary card.
